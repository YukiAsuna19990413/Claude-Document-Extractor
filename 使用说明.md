# 文档提取工具 - 使用说明

## 📋 功能概述
这是一个用于从 VMware Horizon Client 等云桌面环境中自动提取文档的工具。支持通过截图和 OCR 技术识别文档内容。

## 🔧 运行方法

### 方法1：直接运行 Python 脚本
```bash
cd "/Users/haochen/Cursor project"
python3 doc_extractor_gui.py
```

### 方法2：使用启动脚本
```bash
cd "/Users/haochen/Cursor project"
./start_gui.sh
```

## ✨ 主要功能

### 1. 截图区域选择
- **全屏**：捕获整个屏幕
- **左半屏**：捕获左半部分屏幕
- **右半屏**：捕获右半部分屏幕
- **上半屏**：捕获上半部分屏幕
- **下半屏**：捕获下半部分屏幕
- **自定义选择区域**：使用鼠标框选特定区域

### 2. OCR 模型选择
- **API (高精度)**：使用 Zhipu AI API 进行高精度识别
- **本地 Tesseract**：使用本地 Tesseract OCR 引擎

### 3. 配置功能
- **配置 API**：点击打开 API 配置窗口

### 4. 控制功能
- **开始提取**：开始文档提取流程
- **停止**：停止正在进行的提取

### 5. 结果操作
- **复制文本**：将提取的文本复制到剪贴板
- **保存文件**：将提取的文本保存到文件
- **清空日志**：清空提取日志

## 🎯 使用步骤

1. **选择截图区域**
   - 点击"全屏"或其他预设按钮选择区域
   - 或点击"自定义选择区域"，然后在屏幕上拖动鼠标选择

2. **选择 OCR 模型**
   - 如果使用 API：确保已配置 API key
   - 如果使用本地：确保已安装 Tesseract

3. **开始提取**
   - 点击"开始提取"按钮
   - 程序会自动截图并识别文本
   - 结果会显示在日志区域

4. **处理结果**
   - 查看提取的文本
   - 可以复制或保存到文件

## 📋 配置说明

### API 配置
程序会读取 `api_config.json` 文件中的配置：
```json
{
  "api_key": "your_api_key_here",
  "model": "glm-4.5-air"
}
```

### 模型切换
当前支持两种模式：
- `glm-4.5-air`：Zhipu AI 高精度模型
- `tesseract`：本地 Tesseract OCR 引擎

## 🔍 故障排除

### 问题：程序没有显示新功能
**原因**：可能运行的是旧版本
**解决**：
1. 确保在正确的目录运行：`cd "/Users/haochen/Cursor project"`
2. 直接运行：`python3 doc_extractor_gui.py`

### 问题：截图区域选择无效
**原因**：可能是屏幕分辨率问题
**解决**：
1. 尝试不同的预设选项
2. 使用自定义选择功能

### 问题：API 识别失败
**原因**：API key 无效或网络问题
**解决**：
1. 点击"配置 API"重新设置
2. 检查网络连接

## 📝 注意事项

1. 确保 Python 3.7+ 已安装
2. 首次使用前需要安装依赖：
   ```bash
   pip install pillow pyautogui mss pytesseract
   ```
3. 使用 API 模型需要有效的 Zhipu AI API key
4. 确保屏幕上能清晰显示要提取的文档内容
